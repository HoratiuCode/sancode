Write less.
Ship more.

SanCode is a coding standard for AI agents designed to reduce AI-generated code bloat, unnecessary abstractions, and dependency addiction. Its core rule is simple: every line of code must justify its existence.

$ git clone https://github.com/HoratiuCode/sancode.git

Copy CLAUDE.md, CODEX.md, or CURSOR.md into your project. No package. No runtime. No framework.

Principles

Delete Before Create

First look for code to remove, reuse, or collapse. New files are a last resort, not a reflex.

Native Before Package

Use browser APIs, framework defaults, SQL, the filesystem, and the standard library before adding another dependency.

Simple Before Abstract

One known use case does not need providers, managers, adapters, event buses, factories, or architecture theater.

Single File Before Architecture

Keep behavior close together until separation makes the current code easier to understand.

Working Before Perfect

Ship the path users need now. Polish internals after usage proves the feature deserves more time.

Revenue Before Scale

Validate demand, charge money, and talk to users before preparing infrastructure for imaginary traffic.

Decision Tree

Does this already exist? Can I reuse it? Can I simplify it? Can I remove it? Will users care? Will users pay? Can I launch today?

Startup Mode

Before building, ask whether the idea can be validated without code, sold before development, or tested with a manual workflow this week.

Infrastructure Mode

Before adding Redis, Kubernetes, microservices, queues, workers, or search clusters, prove SQLite, cron, native search, and a single server cannot handle the current product.

SanScore

Every task ends with a score out of 100 based on packages added, files created, lines written, complexity introduced, and infrastructure added. Higher means simpler.
