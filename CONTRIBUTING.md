# Contributing to SanCode

SanCode contributions should make AI agents write less code, not more.

## What Belongs Here

Good contributions:

- Clear agent rules
- Practical examples
- Smaller workflows
- Better decision trees
- Real before/after examples from product work
- Founder-focused prompts that prevent overbuilding

Bad contributions:

- Frameworks
- CLIs
- Package managers
- Build systems
- Abstract manifest formats
- Complex governance
- Anything that makes SanCode feel like software instead of a standard

## Contribution Standard

Before opening a pull request, ask:

- Does this reduce code bloat?
- Does this help founders ship faster?
- Does this prevent unnecessary dependencies?
- Does this make AI agents more practical?
- Could this be shorter?

If the answer is no, delete it.

## Style

Write like a founder with scars:

- Direct
- Practical
- Opinionated
- Minimal
- Anti-bloat

Avoid academic language, enterprise process, and soft advice.

## Pull Request Checklist

- No new tooling unless absolutely necessary
- No generated boilerplate
- No vague principles without examples
- No abstractions for future contributors
- Examples compare bad AI output against a simpler SanCode answer

End your pull request description with:

```text
SanScore: X/100
```
